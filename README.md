# 💰 Financial Assistant Bot

Telegram-бот для анализа личных расходов. Принимает CSV-файл с транзакциями и формирует подробный финансовый отчёт на основе фиксированного бюджета.

## 🚀 Что умеет бот

- Принимает CSV-файл с полями `date`, `category`, `amount`
- Фильтрует транзакции за текущий календарный месяц
- Исключает категорию `Зарплата` из расчётов
- Разделяет расходы на **основной бюджет** (30 000 руб.) и **резерв** (10 000 руб.)
- Рассчитывает рекомендуемый дневной лимит трат
- Строит прогноз расходов к концу месяца
- Доступ к боту защищён паролем

- **Ядро**: `Python 3.13+`, `aiogram 3`
- **Аналитика**: `pandas`
- **База данных**: `PostgreSQL` (SQLAlchemy + asyncpg)
- **Инфраструктура**: `Docker`, `uv`, `Ruff`

## 📥 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/ATAGAEV95/Financial_Assistant.git
cd Financial_Assistant
```

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
TG_TOKEN=your_telegram_bot_token
DATABASE_URL=postgresql+asyncpg://user:password@host:5432/dbname
```

| Переменная | Описание |
| :--- | :--- |
| `TG_TOKEN` | Токен бота из [BotFather](https://t.me/BotFather) |
| `DATABASE_URL` | URL подключения к PostgreSQL |

### 3. Запуск через Docker

```bash
docker build -t financial-assistant .
docker run -d --restart=always --name financial-assistant --env-file .env financial-assistant
```

### 4. Запуск локально

```bash
uv sync
uv run python main.py
```

## ⚙️ Настройка бюджета

Параметры бюджета задаются в `app/core/pandas_parses.py`:

```python
RESERVE_CATEGORIES = {"Машина", "Лекарство", "Подарки"}  # Резервные категории
RESERVE_LIMIT = 10_000   # Лимит резерва (руб.)
MAIN_BUDGET = 30_000     # Основной бюджет (руб.)
```

- **Резервные категории** — расходы, которые не входят в основной бюджет (непредвиденные траты, автомобиль, лекарства).
- **Основной бюджет** — лимит на все регулярные расходы за месяц.

### Формат CSV-файла

Бот ожидает файл с заголовками и тремя колонками:

```csv
date,category,amount
01/04/2026,Еда,350
02/04/2026,Кафе,800
03/04/2026,Зарплата,50000
```

- `date` — дата в формате `DD/MM/YYYY`
- `category` — название категории (строка `Зарплата` исключается из расчётов)
- `amount` — сумма (числовое значение, знак не важен — берётся абсолютное значение)
